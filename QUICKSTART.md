## 🎮 第三步: 使用答题功能检验学习效果 (1分钟)

完成一个知识点后,立即进行自测:

### 如何使用QuizWidget

1. 访问 [交互式答题](/tools/quiz)
2. 选择题目类别:
   - **LLM理论基础** - 测试概念理解
   - **Prompt工程** - 测试实战技巧
   - **RAG架构** - 测试系统设计能力
   - **随机抽题** - 综合测试

3. 答题流程:
   ```
   选择答案 → 提交 → 查看解析 → 记录错题
   ```

4. 功能说明:
   - ✅ **即时反馈** - 提交后立即显示正确答案
   - 📊 **详细解析** - 每道题都有知识点讲解
   - 📝 **错题本** - 自动记录答错的题目
   - 🔄 **重复练习** - 可针对错题反复练习

**示例题目**:
```
Q: Transformer架构中的Self-Attention机制主要解决什么问题?
A. 并行计算效率
B. 长距离依赖建模  ← 正确答案
C. 梯度消失问题
D. 过拟合问题

解析: Self-Attention通过计算序列中所有位置的注意力权重,
     能够捕捉长距离依赖关系,这是RNN/LSTM的痛点。
```

## ❓ 常见问题FAQ

### Q1: 我没有AI基础,能直接学习吗?

**A**: 完全可以! LearnPlace专为Java后端开发者设计,从LLM基础概念开始讲解,不需要机器学习背景。建议按顺序学习,不要跳步。

### Q2: 学习过程中遇到问题怎么办?

**A**: 
1. 首先查阅对应章节的[官方文档](/resources/docs)
2. 在[GitHub Issues](https://github.com/your-org/learn-place/issues)中搜索类似问题
3. 在[GitHub Discussions](https://github.com/your-org/learn-place/discussions)中提问
4. 加入社区Discord/微信群交流

### Q3: 需要付费才能使用OpenAI API吗?

**A**: 是的,OpenAI API需要付费使用(约$0.002/1K tokens)。但你可以:
- 使用Ollama本地运行开源模型(免费)
- 申请OpenAI新用户免费额度($5)
- 使用阿里云百炼等国内服务商(有免费额度)

### Q4: 学完这个平台能找到AI相关工作吗?

**A**: LearnPlace提供的是系统化学习路径和实战项目经验,但最终就业还需要:
- 深入理解底层原理(建议补充阅读论文)
- 参与真实项目开发(积累生产环境经验)
- 准备面试题库(本平台提供65+道面试题)
- 持续跟进最新技术(AI领域更新快)

### Q5: 如何贡献内容?

**A**: 欢迎贡献!详见[贡献指南](https://github.com/your-org/learn-place/blob/main/README.md#-贡献指南):
- 修正错别字或链接错误
- 补充新的学习内容
- 优化UI交互体验
- 新增实用工具组件

### Q6: 移动端体验如何?

**A**: LearnPlace采用响应式设计,支持手机/平板/桌面全尺寸屏幕。建议使用Chrome/Safari浏览器获得最佳体验。

### Q7: 可以离线学习吗?

**A**: 目前不支持完全离线,但你可以:
- 克隆GitHub仓库到本地
- 运行 `npm run dev` 在本地浏览
- 下载PDF版本的学习资料(计划中)

### Q8: 学习周期太长,能否加速?

**A**: 12周是建议节奏(每天2小时),你可以根据自身情况调整:
- **加速版** (6周): 每天4小时,跳过已掌握的内容
- **标准版** (12周): 每天2小时,循序渐进
- **休闲版** (24周): 每天1小时,业余时间学习

## 🎓 学习建议总结

### ✅ 应该做的

1. **循序渐进** - 按6阶段顺序学习,不要跳步
2. **理论结合实践** - 每学一个知识点,立即动手写代码
3. **定期自测** - 使用QuizWidget检验学习效果
4. **做笔记** - 建立个人知识库,记录关键概念
5. **参与社区** - 在GitHub Discussions中交流问题
6. **持续打卡** - 保持学习连续性,避免中断

### ❌ 应该避免的

1. **只看不练** - 不要只阅读文档,要动手写代码
2. **死记硬背** - 理解原理比记忆API更重要
3. **追求完美** - 先完成再完善,迭代式学习
4. **孤立学习** - 积极提问和交流,避免闭门造车
5. **忽视基础** - LLM基础和Prompt工程是核心,不要跳过
