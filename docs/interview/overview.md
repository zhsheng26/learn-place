# 面试题库使用说明

本题库包含60+精选问题,覆盖LLM理论、Prompt工程、RAG架构、Agent开发等核心知识点。

- Tokenization
- 解码策略
- KV Cache
- Hallucination

**适合人群**: 所有学习者  
**难度分布**: 简单4题 / 中等5题 / 困难3题

**难度分布**: 简单4题 / 中等5题 / 困难3题

### 3. RAG架构 (15题)
- RAG原理
- 向量数据库
- 文本分块
- 混合搜索
- Query Rewriting
- Rerank重排序
- 长文本处理

**适合人群**: 完成Phase 4的学习者  
**难度分布**: 简单5题 / 中等6题 / 困难4题

### 4. Agent开发 (15题)
- ReAct框架
- 多Agent协作
- Function Calling
- Plan-and-Execute

**适合人群**: 完成Phase 5的学习者  
**难度分布**: 简单4题 / 中等6题 / 困难5题

### 5. Spring AI框架 (10题)
- ChatClient API
- Function Calling
- Vector Store集成
- Streaming Response
- 生产最佳实践

**适合人群**: 学习Spring AI框架的学习者

### 6. 系统设计 (8题)
- 高并发RAG系统
- 智能客服架构
- 多Agent容错设计
- 成本控制架构

**适合人群**: 准备高级职位面试的学习者  
**难度分布**: 困难8题(均为高级设计题)

## 🎯 使用方法

### 方式1: 按分类学习

1. 选择要练习的分类(建议从LLM理论开始)
2. 逐题作答,查看解析和评分标准
3. 记录错题,针对性复习
4. 目标正确率: ≥80%

### 方式2: 模拟考试

**模拟笔试模式**:
1. 随机抽取20道题(涵盖多个分类)
2. 限时60分钟完成
3. 对照答案自评得分
4. 分析薄弱环节

**评分等级**:
- ≥90分: 优秀,可以参加面试
- 75-89分: 良好,继续巩固
- 60-74分: 合格,需系统复习
- <60分: 待提升,重新学习

## 🎤 面试技巧

### 理论知识题
**回答模板**:
1. **定义**: 先给出概念定义
2. **原理**: 解释工作原理
3. **应用**: 说明实际应用场景
4. **对比**: 与其他技术对比优劣

**示例**: "什么是Self-Attention?"
```
Self-Attention是Transformer的核心组件(定义)。
它通过计算序列中所有位置的注意力权重,捕捉长距离依赖关系(原理)。
在机器翻译中,它能对齐源语言和目标语言的对应词汇(应用)。
相比RNN,它能并行计算且不受距离限制(对比)。
```

### 实践应用题
**回答模板**:
1. **场景**: 描述业务场景
2. **方案**: 提出技术方案
3. **理由**: 解释选择理由
4. **权衡**: 讨论trade-off

**示例**: "如何设计一个RAG系统?"
```
场景: 构建企业知识库问答系统
方案: 使用Milvus存储向量,Elasticsearch做关键词搜索,混合检索
理由: Milvus支持高效相似度搜索,ES擅长精确匹配,两者结合提升召回率
权衡: 增加了系统复杂度,但显著提升检索质量,值得投入
```

### 系统设计题
**回答模板**:
1. **需求分析**: 明确功能和非功能需求
2. **架构设计**: 画出系统架构图
3. **核心组件**: 详细说明关键组件
4. **扩展性**: 讨论如何扩展和优化
