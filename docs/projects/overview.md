# 练手项目总览

通过实战项目将理论知识转化为实践能力。5个项目难度递进,覆盖AI应用开发的核心场景。

## 📊 项目对比

| 项目 | 难度 | 时长 | 核心技能 | 技术栈 |
|------|------|------|---------|--------|
| [L1: 智能问答机器人](/projects/project-1-qa-bot) | ⭐ | 1周 | Spring AI基础 | Spring Boot + OpenAI |
| [L2: RAG知识库](/projects/project-2-rag-kb) | ⭐⭐ | 2周 | RAG全链路 | Milvus/PgVector |
| [L2: 代码生成Agent](/projects/project-3-code-agent) | ⭐⭐ | 2周 | Function Calling | GitHub API |
| [L3: 多Agent协作系统](/projects/project-4-multi-agent) | ⭐⭐⭐ | 3周 | Agent架构设计 | Redis + RabbitMQ |
| [L3: 智能客服系统](/projects/project-5-customer-service) | ⭐⭐⭐ | 3周 | 企业级架构 | Elasticsearch + WebSocket |

## 🎯 学习路径建议

### 初学者路线
```
智能问答机器人 → RAG知识库 → 代码生成Agent
```
**目标**: 掌握AI应用开发基础,能够独立构建简单的AI应用

### 进阶路线
```
智能问答机器人 → RAG知识库 → 多Agent协作系统 → 智能客服系统
```
**目标**: 具备企业级AI应用架构能力,能够设计复杂系统

## 💡 项目学习方法

### 1. 理解需求
仔细阅读项目需求和业务场景,明确要实现的功能。

### 2. 学习架构
研究项目架构图,理解各个组件的职责和交互方式。

### 3. 分步实现
按照实施步骤逐步编码,每完成一步都进行测试。

### 4. 遇到问题
- 查看"常见问题"部分
- 在GitHub Issues中提问
- 参考完整代码示例

### 5. 扩展优化
完成基本要求后,尝试实现"扩展功能",提升项目质量。

## 🛠️ 技术准备

### 必备知识
- ✅ Java 17+ 和 Spring Boot 3
- ✅ Maven或Gradle构建工具
- ✅ RESTful API设计
- ✅ Git版本控制

### 推荐知识
- ⭐ Docker容器化部署
- ⭐ 数据库基础(MySQL/PostgreSQL)
- ⭐ Redis缓存使用
- ⭐ 消息队列(RabbitMQ/Kafka)

## 🚀 开始第一个项目

建议从 [L1: 智能问答机器人](/projects/project-1-qa-bot) 开始,这是最简单的入门项目。

**预计时间**: 1周  
**前置要求**: 
- 熟悉Spring Boot基础
- 了解HTTP协议
- 有OpenAI API密钥(或使用其他LLM提供商)

**学习目标**:
- 掌握Spring AI的基本使用
- 理解ChatClient的API设计
- 学习会话管理技巧
- 实践Prompt工程基础

## 📤 项目提交流程

完成项目后,可以提交你的成果:

1. **代码托管**: 将项目代码上传到GitHub
2. **填写表单**: 在[进度追踪](/tools/progress)页面提交GitHub链接
3. **自我评估**: 对照验收标准自评
4. **获取反馈**: 社区成员会Review你的代码

**提交检查清单**:
- [ ] 代码可正常运行
- [ ] 包含README说明文档
- [ ] 通过所有验收标准
- [ ] 添加必要的注释
