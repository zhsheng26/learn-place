# LearnPlace 核心组件测试指南

本文档提供LearnPlace项目三个核心Vue组件和相关工具页面的完整测试说明。

## 📋 测试前准备

### 1. 启动开发服务器
```bash
npm run dev
```

访问 `http://localhost:5173/learn-place/` (根据实际端口调整)

### 2. 清除浏览器缓存
在测试前建议清除localStorage数据,确保从头开始测试:
```javascript
// 在浏览器控制台执行
localStorage.clear()
```

## 🐛 常见问题排查

### 问题1: Mermaid图表不显示
**可能原因**:
- Mermaid库未正确加载
- 图表定义语法错误

**排查步骤**:
1. 检查浏览器控制台是否有错误
2. 确认 `package.json` 中包含 `"mermaid": "^10.6.0"`
3. 运行 `npm install` 重新安装依赖

### 问题2: ECharts雷达图空白
**可能原因**:
- ECharts库未正确初始化
- chartRef元素未挂载

**排查步骤**:
1. 检查控制台是否有 "echarts is not defined" 错误
2. 确认 `package.json` 中包含 `"echarts": "^5.4.0"`
3. 检查 `onMounted` 是否正确执行

### 问题3: localStorage数据不持久化
**可能原因**:
- 浏览器隐私模式
- localStorage被禁用

**排查步骤**:
1. 退出隐私模式
2. 检查浏览器设置中是否启用了localStorage
3. 尝试在其他浏览器测试

### 问题4: 组件样式异常
**可能原因**:
- VitePress主题变量未正确应用
- CSS作用域问题

**排查步骤**:
1. 检查是否使用了 `scoped` 样式
2. 确认VitePress主题正常运行
3. 检查浏览器开发者工具中的CSS优先级
